# 강의 보관/복원 운영 가이드

이 문서는 이 사이트를 오래 운영할 때, 강의를 추가하거나 보관하거나 다시 복원할 때 참고하는 간단한 안내서입니다.

## 기본 원칙

- GitHub Pages에는 최근 강의만 둡니다.
- 오래된 강의는 프로젝트에서 빼서 용량을 줄입니다.
- 오래된 강의를 빼기 전에 반드시 `수정 반영 JSON`을 먼저 다운로드합니다.
- 나중에 다시 보고 싶으면 `슬라이드 이미지 폴더 + 수정 반영된 results.json + lectures.json 항목`을 다시 넣으면 됩니다.

## 강의 1개가 구성되는 방식

강의 1개는 보통 아래 3가지로 생각하면 됩니다.

- 슬라이드 이미지 폴더
- results.json
- lectures.json 안의 강의 연결 정보

예시:

- `slides/`
- `results.json`
- `lectures.json` 안의 `resultsPath`, `slidesPath`

## 강의를 새로 추가할 때

1. 새 강의의 슬라이드 이미지 폴더를 준비합니다.
2. 새 강의의 `results.json`을 준비합니다.
3. `lectures.json`에 과목/강의 정보를 추가합니다.
4. `resultsPath`와 `slidesPath`를 실제 파일 위치에 맞게 적습니다.
5. 로컬에서 사이트를 열어 과목 목록 > 강의 목록 > 슬라이드 목록이 잘 보이는지 확인합니다.

## 강의를 보관할 때

1. 사이트에서 그 강의를 열어 수정 내용이 필요한 만큼 반영되었는지 확인합니다.
2. 강의 목록 화면에서 `수정 반영 JSON` 버튼을 눌러 최종 JSON을 다운로드합니다.
3. 아래 2가지를 같은 보관 폴더에 따로 저장합니다.
   - 슬라이드 이미지 폴더
   - 다운로드한 수정 반영 JSON
4. 프로젝트에서는 그 강의용 이미지 폴더와 JSON 파일을 제거합니다.
5. `lectures.json`에서도 그 강의 항목을 제거합니다.
6. 로컬에서 사이트를 다시 열어 해당 강의가 목록에서 빠졌는지 확인합니다.

## 강의를 다시 복원할 때

1. 보관해둔 슬라이드 이미지 폴더를 프로젝트 안으로 다시 넣습니다.
2. 보관해둔 수정 반영 JSON 파일을 적절한 위치에 다시 넣고 파일명을 정리합니다.
3. `lectures.json`에 그 강의 항목을 다시 추가합니다.
4. `resultsPath`와 `slidesPath`를 실제 위치에 맞게 적습니다.
5. 로컬에서 사이트를 열어 과목 > 강의 > 슬라이드 > 상세 보기가 정상인지 확인합니다.

## Supabase와 관련된 점

- Supabase에는 수정된 설명/복습/용어 데이터가 저장됩니다.
- viewer에서 각 항목 저장 버튼을 누르면 자동으로 Supabase에도 반영됩니다.
- 하지만 보관용 기준 파일은 여전히 `수정 반영 JSON 다운로드본`을 따로 가지고 있는 것이 가장 안전합니다.

## 가장 안전한 백업 세트

강의 하나를 완전히 다시 살릴 수 있으려면 아래 2가지는 꼭 필요합니다.

- 그 강의의 슬라이드 이미지들
- 그 강의의 수정 반영된 results.json

여기에 `lectures.json`의 해당 항목 정보까지 같이 메모해두면 복원이 더 쉽습니다.

## 작업 후 꼭 확인할 것

- 과목 목록이 정상인지
- 강의 목록이 정상인지
- 슬라이드 상세가 열리는지
- 설명/복습/용어 수정이 저장되는지
- `수정 반영 JSON` 다운로드가 되는지
